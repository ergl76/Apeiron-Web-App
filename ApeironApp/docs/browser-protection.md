## Technische Details

### beforeunload Event
- **Standard:** HTML5 Specification
- **Support:** Alle modernen Browser
- **Limitation:** Custom-Messages werden von Browsern ignoriert (Security seit Chrome 51)
- **Verhalten:** Browser zeigt generische Warnung ("√Ñnderungen werden m√∂glicherweise nicht gespeichert")

### popstate Event
- **Standard:** HTML5 History API
- **Technik:** History Manipulation verhindert Navigation
- **Alternative:** Kein `e.preventDefault()` m√∂glich bei popstate
- **L√∂sung:** Neuer History Entry wird sofort gepusht (verhindert tats√§chliche Navigation)

### Touch Events mit passive: false
```javascript

```
- **passive: false** - **KRITISCH** f√ºr `preventDefault()` Funktionalit√§t
- **Default:** `passive: true` (Performance-Optimierung)
- **Warum false?** Browser erlaubt `preventDefault()` nur wenn explizit deklariert
- **Trade-off:** Minimale Performance-Reduktion f√ºr Schutz-Funktionalit√§t

### Module-Level Flag
```javascript

```
- **Grund:** Shared State zwischen useEffect Cleanup und Event Handlers
- **Problem gel√∂st:** Cleanup deaktiviert Schutz auch wenn Component unmounted
- **Alternative:** useState w√ºrde bei Cleanup nicht funktionieren (React Closure)

### CSS overscroll-behavior

- **Standard:** CSS Overscroll Behavior Module Level 1
- **Support:** Chrome 63+, Firefox 59+, Safari 16+
- **Fallback:** Touch-Events f√ºr √§ltere Browser
- **Warum beides?** Defense in Depth - CSS als Primary, Touch als Fallback
